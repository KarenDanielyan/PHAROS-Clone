📖 Overview

The PHAROS Virtual Laser Clone provides a complete simulation of the PHAROS laser REST API for development purposes. This virtual clone implements all endpoints with identical behavior to the original hardware API.

🎯 Purpose: This virtual clone is designed for developing and testing laser automation applications without requiring physical hardware access.

API Structure

🔧 Basic API (/v1/Basic)

Essential laser operations including power control, state management, and output control. Maps 1:1 with the UI title screen.

⚙️ Advanced API (/v1/Advanced)

Complex functionality including detailed state information, hardware configuration, and advanced preset management.

🔬 Raw API (/v1/Raw)

Low-level functionality for advanced users only, including direct wrapper function execution.

Authentication & Security

⚠️ Security Note: Like the real PHAROS laser, this API operates on localhost only and requires no authentication.

🔄 Laser State Machine

The virtual laser maintains a realistic state machine that mirrors the behavior of the physical PHAROS laser system.

State Transition Flow

StateDisconnected
→
StateOff
→
StateOperational
→
StateEmissionOn

Available States

  • StateDisconnected: Laser is not connected or available
  • StateOff: Laser is connected but turned off
  • StateStandingBy: Laser is in standby mode
  • StateOperational: Laser is ready for operation
  • StateEmissionOn: Laser output is enabled and active
  • StateFailure: Laser has encountered an error
  • StateShuttingDown: Laser is in shutdown process

🔧 Basic API Endpoints

GET /v1/Basic Get all basic properties in batch

Returns all basic laser properties that have GET methods in a single response. Convenient for getting complete laser status.

{ "ActualAttenuatorPercentage": 50.1, "ActualHarmonic": 1, "ActualOutputEnergy": 50.0, "ActualOutputFrequency": 100.1, "ActualOutputPower": 2.501, "ActualPpDivider": 1, "ActualRaFrequency": 100.1, "ActualRaPower": 5.0, "ActualStateName": "StateOperational", "GeneralStatus": "Operational", "IsOutputEnabled": false, "SelectedPresetIndex": 1, "TargetAttenuatorPercentage": 50.0, "TargetPpDivider": 1 }
GET /v1/Basic/ActualOutputPower Get current laser output power

Returns the actual laser output power in watts, taking PP divider and attenuator into account.

"2.501"
GET /v1/Basic/ActualStateName Get current laser state

Returns the name of the current internal state machine state.

"StateOperational"
GET /v1/Basic/IsOutputEnabled Check if laser output is enabled

Returns whether the laser output is currently enabled.

false
POST /v1/Basic/TurnOn Turn on the laser

Transitions the laser from Off or Disconnected state to Operational state.

⚠️ State Requirement: Can only be called when laser is in StateOff or StateDisconnected.
{ "status": "success", "message": "Laser turned on successfully" }
POST /v1/Basic/EnableOutput Enable laser output

Enables laser output and transitions to StateEmissionOn.

⚠️ State Requirement: Can only be called when laser is in StateOperational.
{ "status": "success", "message": "Output enabled successfully" }
PUT /v1/Basic/TargetAttenuatorPercentage Set attenuator percentage

Sets the target attenuator percentage (0-100%).

Body: number (0-100)
Example: "75.5"
{ "status": "success", "message": "Attenuator percentage set to 75.5%" }

PowerShell Examples (Windows)

# Get laser status Invoke-RestMethod -Uri "http://localhost:20020/v1/Basic" -Method Get # Set attenuator percentage Invoke-RestMethod -Uri "http://localhost:20020/v1/Basic/TargetAttenuatorPercentage" ` -Method Put -Body "50.0" -ContentType "application/json" # Turn on laser Invoke-RestMethod -Uri "http://localhost:20020/v1/Basic/TurnOn" -Method Post

Python Example

import requests # Base URL for the virtual laser base_url = "http://localhost:20020/v1" # Get all basic properties response = requests.get(f"{base_url}/Basic") laser_status = response.json() print(f"Laser state: {laser_status['ActualStateName']}") # Set attenuator percentage requests.put(f"{base_url}/Basic/TargetAttenuatorPercentage", data="75.0") # Enable output if laser is operational if laser_status['ActualStateName'] == 'StateOperational': requests.post(f"{base_url}/Basic/EnableOutput") print("Laser output enabled")

❌ Error Handling

The API returns appropriate HTTP status codes and error messages for invalid operations.

Error Code Guidelines:
  • 400 Bad Request: Invalid parameter values, malformed requests, or validation errors
  • 403 Forbidden: Operation is forbidden due to current laser state (e.g., cannot enable output when laser is off)
  • 404 Not Found: Endpoint does not exist

Common Error Responses

400 Bad Request Invalid parameter format or value
{ "error": { "code": 400, "message": "Attenuator percentage must be between 0 and 100", "timestamp": "2025-09-06T10:30:00.000Z" } }
403 Forbidden Operation forbidden in current laser state
{ "error": { "code": 403, "message": "Cannot enable output in state StateStandingBy", "timestamp": "2025-09-06T10:30:00.000Z" } }
404 Not Found Endpoint does not exist
{ "detail": "Not Found" }
